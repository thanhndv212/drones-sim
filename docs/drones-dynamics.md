# Drone Dynamics

## Introduction
Drone dynamics involves the study of the forces and motions that govern the behavior of drones during flight. Understanding these dynamics is crucial for designing, controlling, and optimizing drone performance in various applications.

## Key Concepts

### 1. Degrees of Freedom (DoF)
- Drones typically operate with six degrees of freedom:
  - **Translational Motion**: Forward/backward, left/right, up/down.
  - **Rotational Motion**: Roll, pitch, yaw.

### 2. Forces Acting on a Drone
- **Thrust**: Generated by propellers to counteract gravity and enable lift.
- **Gravity**: Acts downward due to the drone's weight.
- **Drag**: Air resistance opposing motion.
- **Lift**: Generated by aerodynamic surfaces (if present).

### 3. Moments and Torques
- **Roll**: Rotation around the longitudinal axis.
- **Pitch**: Rotation around the lateral axis.
- **Yaw**: Rotation around the vertical axis.

### 4. Equations of Motion
- Governed by Newton-Euler equations:
  - Translational motion: `F = ma`
  - Rotational motion: `τ = Iα`

### 5. Stability and Control
- **Static Stability**: Ability to return to equilibrium after a disturbance.
- **Dynamic Stability**: Behavior over time after a disturbance.
- **Control Inputs**: Adjustments to propeller speeds to achieve desired motion.

## Drone Types and Dynamics

### 1. Multirotors
- **Quadcopters**: Four rotors, widely used for stability and control.
- **Hexacopters/Octocopters**: More rotors for redundancy and payload capacity.

### 2. Fixed-Wing Drones

## Mathematical Dynamic Model of Quadcopters

The dynamics of quadcopters are governed by Newton-Euler equations, which describe the translational and rotational motion of the drone. Below is the mathematical model:

### 1. Translational Dynamics
The translational motion is described by:

$$
\begin{aligned}
    m \ddot{x} &= -u_1 \sin(\phi) \sin(\psi) + u_1 \cos(\phi) \sin(\theta) \cos(\psi) - D_x, \\
    m \ddot{y} &= u_1 \sin(\phi) \cos(\psi) + u_1 \cos(\phi) \sin(\theta) \sin(\psi) - D_y, \\
    m \ddot{z} &= u_1 \cos(\phi) \cos(\theta) - mg - D_z,
\end{aligned}
$$

where:
- $m$: Mass of the quadcopter.
- $\ddot{x}, \ddot{y}, \ddot{z}$: Accelerations in the $x$, $y$, and $z$ directions.
- $u_1$: Total thrust generated by the propellers.
- $\phi, \theta, \psi$: Roll, pitch, and yaw angles.
- $D_x, D_y, D_z$: Drag forces in the $x$, $y$, and $z$ directions.
- $g$: Gravitational acceleration.

### 2. Rotational Dynamics
The rotational motion is described by:

$$
\begin{aligned}
    I_x \ddot{\phi} &= \tau_\phi - (I_y - I_z) \dot{\theta} \dot{\psi}, \\
    I_y \ddot{\theta} &= \tau_\theta - (I_z - I_x) \dot{\phi} \dot{\psi}, \\
    I_z \ddot{\psi} &= \tau_\psi - (I_x - I_y) \dot{\phi} \dot{\theta},
\end{aligned}
$$

where:
- $I_x, I_y, I_z$: Moments of inertia about the $x$, $y$, and $z$ axes.
- $\ddot{\phi}, \ddot{\theta}, \ddot{\psi}$: Angular accelerations.
- $\tau_\phi, \tau_\theta, \tau_\psi$: Torques generated by the propellers.
- $\dot{\phi}, \dot{\theta}, \dot{\psi}$: Angular velocities.

### 3. Thrust and Torque Relationships
The thrust and torques are related to the propeller speeds as follows:

$$
\begin{aligned}
    u_1 &= k_f (\omega_1^2 + \omega_2^2 + \omega_3^2 + \omega_4^2), \\
    \tau_\phi &= k_f L (\omega_2^2 - \omega_4^2), \\
    \tau_\theta &= k_f L (\omega_3^2 - \omega_1^2), \\
    \tau_\psi &= k_m (\omega_1^2 - \omega_2^2 + \omega_3^2 - \omega_4^2),
\end{aligned}
$$

where:
- $k_f$: Thrust coefficient.
- $k_m$: Drag (moment) coefficient.
- $L$: Distance from the center of the quadcopter to the propeller.
- $\omega_1, \omega_2, \omega_3, \omega_4$: Angular velocities of the four propellers.

### 4. State-Space Representation
The above equations can be represented in state-space form for control design:

$$
\dot{X} = AX + BU,
$$

where:
- $X$: State vector $[x, y, z, \phi, \theta, \psi, \dot{x}, \dot{y}, \dot{z}, \dot{\phi}, \dot{\theta}, \dot{\psi}]^T$.
- $U$: Input vector $[u_1, \tau_\phi, \tau_\theta, \tau_\psi]^T$.
- $A$: System matrix.
- $B$: Input matrix.

### 5. Typical Controllers for Quadcopters
Quadcopters are typically controlled using the following types of controllers, each suited for specific control tasks:

1. **PID Controllers**:
   - **Principle**: PID controllers calculate control inputs based on the error between the desired and actual states. The proportional term reacts to the current error, the integral term addresses accumulated past errors, and the derivative term predicts future errors based on the rate of change.
   - **Feedforward Usage**: Feedforward can be used to provide an initial control input based on a model of the system, reducing the burden on the feedback loop and improving response time, especially for altitude and position control.
   - The control input $u$ is given by:
     $$
     u = K_p e(t) + K_i \int e(t) dt + K_d \frac{de(t)}{dt},
     $$
     where:
     - $e(t)$: Error between the desired and actual state.
     - $K_p, K_i, K_d$: Proportional, integral, and derivative gains.

2. **LQR Controllers**:
   - **Principle**: LQR controllers minimize a quadratic cost function that balances state deviations and control effort. They use a system model to compute optimal control inputs.
   - **Feedforward Usage**: Feedforward can be used to handle known disturbances or to follow a predefined trajectory, while the feedback loop corrects deviations from the optimal path.
   - The control input $u$ is given by:
     $$
     u = -Kx,
     $$
     where:
     - $K$: Gain matrix obtained by solving the Riccati equation.
     - $x$: State vector.

    ### Explanation of the Riccati Equation
    The Riccati equation is a key mathematical tool used in optimal control, particularly in Linear Quadratic Regulator (LQR) design. It is used to compute the gain matrix $K$ that minimizes a quadratic cost function, balancing state deviations and control effort.

    **Riccati Equation**: The gain matrix $K$ is computed by solving the Algebraic Riccati Equation (ARE):
        $$
        A^T P + P A - P B R^{-1} B^T P + Q = 0,
        $$
        where:
        - $P$: Symmetric positive-definite matrix.
        - $Q$: State weighting matrix.
        - $R$: Control weighting matrix.
        - $A, B$: System matrices.

    **Purpose**:
    The Riccati equation determines the matrix $P$, which is then used to compute the optimal feedback gain matrix $K$:
    $$
    K = R^{-1} B^T P.
    $$
    This gain matrix $K$ is used in the control law:
    $$
    u = -Kx,
    $$
    where $x$ is the state vector and $u$ is the control input.

    **Intuition**:
    - The matrix $Q$ emphasizes the importance of keeping the system states close to the desired values.
    - The matrix $R$ ensures that the control inputs are not excessively large, balancing performance and energy efficiency.
    - The solution $P$ encapsulates the trade-off between these objectives, leading to an optimal control strategy.

3. **Model Predictive Controllers (MPC)**:
   - **Principle**: MPC predicts future states using a dynamic model and optimizes control inputs over a finite time horizon to achieve desired objectives while respecting constraints.
   - **Feedforward Usage**: Feedforward is inherently part of MPC, as it uses predictions to compute control inputs proactively, reducing reliance on feedback corrections.
   - The control input $u$ is obtained by solving the optimization problem:
     $$
     \min_u \sum_{k=0}^N \left( x_k^T Q x_k + u_k^T R u_k \right),
     $$
     subject to:
     $$
     x_{k+1} = Ax_k + Bu_k,
     $$
     where:
     - $Q, R$: Weighting matrices.
     - $A, B$: System matrices.
     - $x_k, u_k$: State and control input at step $k$.

4. **Adaptive Controllers**:
   - **Principle**: Adaptive controllers adjust their parameters in real-time to account for changes in system dynamics, such as payload variations or environmental disturbances.
   - **Feedforward Usage**: Feedforward can provide baseline control inputs based on estimated system parameters, while the adaptive mechanism refines these inputs as conditions change.
   - The control law adapts parameters $
abla$ in real-time:
     $$
     u = K(t)x,
     $$
     where $K(t)$ is updated based on:
     $$
     \dot{K}(t) = \gamma x e^T,
     $$
     with $\gamma$ being the adaptation rate and $e$ the error.

5. **Sliding Mode Controllers (SMC)**:
   - **Principle**: SMC forces the system to follow a predefined sliding surface by applying discontinuous control actions, ensuring robustness to disturbances and model uncertainties.
   - **Feedforward Usage**: Feedforward can be used to approximate the required control effort to stay on the sliding surface, reducing chattering and improving efficiency.

6. **Neural Network-Based Controllers**:
   - **Principle**: These controllers use machine learning models to approximate the system dynamics and compute control inputs based on learned patterns.
   - **Feedforward Usage**: Feedforward can leverage the neural network's predictions to provide proactive control inputs, while feedback corrects for unmodeled dynamics or errors in the learned model.

7. **Fuzzy Logic Controllers**:
   - **Principle**: Fuzzy logic controllers use linguistic rules and membership functions to handle uncertainties and imprecise inputs, providing smooth and stable control.
   - **Feedforward Usage**: Feedforward can apply rule-based control actions for anticipated conditions, while feedback adjusts for deviations or unexpected changes.

Each controller type is chosen based on the specific requirements of the application, such as precision, robustness, adaptability, and computational complexity.

### State Estimation and Sensor Fusion

#### Overview
State estimation combines data from multiple sensors to accurately determine the drone's position, velocity, and orientation. Common sensors include:
- IMUs (accelerometers, gyroscopes)
- GPS receivers
- Vision sensors (cameras)
- Barometers
- Magnetometers

#### Extended Kalman Filter (EKF)
The EKF is widely used for nonlinear state estimation in drone systems. It extends the standard Kalman filter to nonlinear systems by linearizing the system model around the current state estimate.

##### When to Use EKF
EKF should be used in the following situations:
- When the system dynamics or measurement models are nonlinear (as in drone dynamics)
- When you need to fuse data from multiple sensor sources (IMU, GPS, camera, etc.)
- When state variables can't be measured directly (like orientation from accelerometer and gyroscope)
- When measurements and system dynamics have noise and uncertainty
- When real-time state estimation is required

##### EKF Mathematical Formulation
$$
\begin{aligned}
\text{Prediction:} & \\
\hat{x}_{k|k-1} &= f(\hat{x}_{k-1|k-1}, u_k) \\
P_{k|k-1} &= F_k P_{k-1|k-1} F_k^T + Q_k \\
\\
\text{Update:} & \\
K_k &= P_{k|k-1} H_k^T(H_k P_{k|k-1} H_k^T + R_k)^{-1} \\
\hat{x}_{k|k} &= \hat{x}_{k|k-1} + K_k(z_k - h(\hat{x}_{k|k-1})) \\
P_{k|k} &= (I - K_k H_k)P_{k|k-1}
\end{aligned}
$$

##### Implementation Considerations
- **Jacobian Calculation**: Computing accurate Jacobian matrices (F_k and H_k) is crucial
- **Tuning**: Proper tuning of process noise (Q_k) and measurement noise (R_k) is essential
- **Adaptive Parameters**: Dynamic adjustment of noise parameters can improve performance
- **Numerical Stability**: Joseph form of covariance update can improve numerical stability
- **Initialization**: Good initial state and covariance estimates prevent divergence

### Explanation of EKF Equations

#### Prediction Step
1. **State Prediction**:
   $$
   \hat{x}_{k|k-1} = f(\hat{x}_{k-1|k-1}, u_k)
   $$
   - $\hat{x}_{k|k-1}$: Predicted state estimate at time \(k\) based on the previous state estimate at \(k-1\).
   - $f(\cdot)$: Nonlinear state transition function that models the system dynamics.
   - \(\hat{x}_{k-1|k-1}\): State estimate at time \(k-1\) after the update step.
   - \(u_k\): Control input applied at time \(k\).

2. **Error Covariance Prediction**:
   $$
   P_{k|k-1} = F_k P_{k-1|k-1} F_k^T + Q_k
   $$
   - \(P_{k|k-1}\): Predicted error covariance matrix, representing the uncertainty in the predicted state.
   - \(F_k\): Jacobian matrix of the state transition function \(f(\cdot)\) with respect to the state, evaluated at \(\hat{x}_{k-1|k-1}\).
   - \(P_{k-1|k-1}\): Error covariance matrix at time \(k-1\) after the update step.
   - \(Q_k\): Process noise covariance matrix, representing uncertainty in the system model.

#### Update Step
3. **Kalman Gain**:
   $$
   K_k = P_{k|k-1} H_k^T \left(H_k P_{k|k-1} H_k^T + R_k\right)^{-1}
   $$
   - \(K_k\): Kalman gain, which determines how much the measurements influence the state estimate.
   - \(H_k\): Jacobian matrix of the measurement function \(h(\cdot)\) with respect to the state, evaluated at \(\hat{x}_{k|k-1}\).
   - \(R_k\): Measurement noise covariance matrix, representing uncertainty in the sensor measurements.

4. **State Update**:
   $$
   \hat{x}_{k|k} = \hat{x}_{k|k-1} + K_k \left(z_k - h(\hat{x}_{k|k-1})\right)
   $$
   - \(\hat{x}_{k|k}\): Updated state estimate at time \(k\) after incorporating the measurement.
   - \(z_k\): Measurement vector at time \(k\).
   - \(h(\cdot)\): Nonlinear measurement function that maps the state to the measurement space.
   - \(z_k - h(\hat{x}_{k|k-1})\): Measurement residual (difference between the actual measurement and the predicted measurement).

5. **Error Covariance Update**:
   $$
   P_{k|k} = \left(I - K_k H_k\right) P_{k|k-1}
   $$
   - \(P_{k|k}\): Updated error covariance matrix at time \(k\) after the update step.
   - \(I\): Identity matrix, ensuring proper dimensionality.

#### Advanced EKF Variants for Drone Applications

1. **Adaptive EKF**:
   - Dynamically adjusts process and measurement noise covariances based on innovation statistics
   - Particularly useful for drones operating in changing environments
   - Implementation often monitors innovation sequence to detect changes in system dynamics

2. **Unscented Kalman Filter (UKF)**:
   - Alternative to EKF that uses deterministic sampling points (sigma points) instead of linearization
   - Typically more accurate for highly nonlinear systems
   - Avoids computation of Jacobian matrices but is computationally more intensive

3. **Robust EKF**:
   - Designed to handle outliers and non-Gaussian noise
   - Uses statistical methods to detect and reject erroneous measurements
   - Critical for drone operations in GPS-denied or electromagnetically noisy environments

4. **Dual EKF**:
   - Simultaneously estimates states and parameters
   - Useful for drones with changing dynamics (e.g., payload changes, battery depletion)
   - Employs two interconnected EKFs: one for state estimation and one for parameter estimation

### Gimbal Modeling and Control

#### Gimbal Dynamics
Gimbals are modeled as multi-degree-of-freedom (DOF) systems with inertia, damping, and external disturbances like wind. The dynamics are described by:
$$I \ddot{\theta} + b \dot{\theta} + k \theta = \tau,$$
where:
- $I$: Moment of inertia.
- $b$: Damping coefficient.
- $k$: Stiffness.
- $\tau$: Applied torque.

#### Stabilization
Feedback control compensates for drone motion to keep the camera steady. Controllers like PID or LQR are commonly used.

#### Actuator Control
Gimbal actuators are typically brushless motors controlled via PWM or field-oriented control (FOC). Control involves managing torque, speed, and current.

### Sensor Fusion

#### IMU (Inertial Measurement Unit)
IMUs measure acceleration and angular velocity but suffer from errors like bias, noise, and drift.

#### Magnetometer
Magnetometers provide heading information but are sensitive to magnetic disturbances from motors or metal parts.

#### Sensor Fusion Algorithms
Algorithms like complementary filters, Kalman filters, and extended Kalman filters (EKF) combine data from IMUs, magnetometers, and other sensors to improve accuracy.

### Signal Processing

#### Aliasing
Aliasing occurs when the sampling frequency is too low. Anti-aliasing filters and proper sampling rates (following the Nyquist rule) are used to prevent it.

#### Filtering
Low-pass filters, such as Butterworth filters, smooth noisy sensor signals. Zero-phase filtering (e.g., `filtfilt()` in MATLAB) is often used to avoid phase distortion.

#### Advanced Signal Processing Techniques

##### Digital Filters for Quadcopters

1. **Low-Pass Filters**
   - **Butterworth**: Maximally flat magnitude response, widely used for smoothing sensor data
   ```matlab
   [b, a] = butter(2, 0.1); % 2nd order, normalized cutoff at 0.1
   filtered_signal = filter(b, a, raw_signal);
   ```
   - **Chebyshev**: Steeper roll-off but ripple in passband/stopband
   - **Elliptic**: Sharpest transition but ripples in both passband and stopband

2. **Kalman Filtering as DSP**
   ```cpp
   // One-dimensional Kalman filter implementation
   float KalmanFilter1D(float measurement, float &state, float &p, float q, float r) {
       // Prediction
       p = p + q;
       
       // Update
       float k = p / (p + r);  // Kalman gain
       state = state + k * (measurement - state);
       p = (1 - k) * p;
       
       return state;
   }
   ```

3. **Complementary Filters**
   - Simple yet effective solution to combine high-frequency gyro data with low-frequency accelerometer data
   ```python
   def complementary_filter(accel_angle, gyro_rate, prev_angle, dt, alpha=0.98):
       return alpha * (prev_angle + gyro_rate * dt) + (1 - alpha) * accel_angle
   ```

##### Frequency Domain Analysis

1. **Fast Fourier Transform (FFT)**
   Used to analyze vibration signatures and identify mechanical resonances:
   ```matlab
   Fs = 1000;                    % Sampling frequency (Hz)
   signal = imu_accel_data(:,1); % X-axis acceleration
   N = length(signal);           % Signal length
   
   Y = fft(signal);              % Compute FFT
   P2 = abs(Y/N);                % Two-sided spectrum
   P1 = P2(1:N/2+1);             % Single-sided spectrum
   P1(2:end-1) = 2*P1(2:end-1);  % Account for energy
   
   f = Fs*(0:(N/2))/N;           % Frequency vector
   plot(f, P1);                  % Plot the spectrum
   ```

2. **Wavelet Analysis**
   Better for transient vibrations and motor anomaly detection:
   ```matlab
   [wt, f] = cwt(motor_current, Fs); % Continuous wavelet transform
   ```

##### Practical Implementations

1. **Moving Average Filter**
   Simple and computationally efficient:
   ```cpp
   // Simple moving average implementation
   float moving_average(float new_value, float *buffer, int buffer_size) {
       static int index = 0;
       static float sum = 0;
       
       // Subtract the oldest value
       sum -= buffer[index];
       // Add the new value
       buffer[index] = new_value;
       sum += new_value;
       
       // Move to next position
       index = (index + 1) % buffer_size;
       
       // Return the average
       return sum / buffer_size;
   }
   ```

2. **Notch Filters**
   Used to eliminate specific frequencies like motor vibrations:
   ```matlab
   % Create notch filter for motor frequencies (e.g., 80 Hz)
   motor_freq = 80;

   Fs = 1000;     % Sampling frequency

   bw = wo/Q;
   ```
